E-Commerce Lengkap

  • Sistem keranjang belanja

Responsive Design

3. Setup Firebase Lengkap

Firebase adalah backbone dari aplikasi Femmeia Store. Ikuti panduan step-by-step berikut untuk setup yang sempurna.

üöÄ 3.1 Membuat Project Firebase

  1. Buka Firebase Console dan login dengan akun Google
  2. Klik tombol "Add project" atau "Tambah project"
  3. Masukkan nama project yang unik (contoh: Femmeia-store-2025)
  4. Pilih apakah ingin mengaktifkan Google Analytics (recommended: Ya)
  5. Jika mengaktifkan Analytics, pilih atau buat akun Analytics
  6. Klik "Create project" dan tunggu proses selesai
  7. Setelah selesai, klik "Continue" untuk masuk ke dashboard

Peringatan Nama Project

Nama project Firebase harus unik secara global. Jika nama yang Anda pilih sudah digunakan, tambahkan tahun atau nomor di belakangnya.

üîê 3.2 Setup Authentication

  1. Di dashboard Firebase, klik "Authentication" di menu kiri
  2. Klik "Get started" jika ini pertama kali
  3. Pilih tab "Sign-in method"
  4. Klik pada "Email/Password"
  5. Toggle "Enable" untuk mengaktifkan
  6. Opsional: Aktifkan juga "Email link (passwordless sign-in)"
  7. Klik "Save"

üîß Konfigurasi Advanced Authentication:

üóÑÔ∏è 3.3 Setup Firestore Database

  1. Klik "Firestore Database" di menu kiri
  2. Klik "Create database"
  3. Pilih "Start in test mode" (untuk development)
  4. Pilih lokasi server terdekat dengan target user Anda:
    • asia-southeast1 (Singapore) - Untuk Indonesia
    • asia-southeast2 (Jakarta) - Untuk Indonesia (jika tersedia)
  5. Klik "Done" dan tunggu database dibuat

8. Optimasi Performa

Website Femmeia Store dirancang untuk performa cepat. Berikut adalah beberapa aspek penting:

üñºÔ∏è 8.1 Lazy Loading Gambar

Gambar produk dimuat secara "lazy" (hanya saat dibutuhkan) untuk mempercepat waktu muat awal halaman.

üí° Cara Kerja Lazy Loading

Gambar produk di `index.html` menggunakan atribut `data-lazy` untuk URL gambar. JavaScript akan mengganti `data-lazy` menjadi `src` hanya ketika gambar masuk ke viewport pengguna. Pastikan Anda menggunakan format ini untuk gambar produk.

‚ö° 8.2 Preload Critical Resources

Font dan resource penting lainnya di-preload untuk memastikan pengalaman pengguna yang cepat.

Preload Links di index.html
// Bagian script di index.html (sekitar baris 3200) const preloadLinks = [ 'https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap' ]; preloadLinks.forEach(href => { const link = document.createElement('link'); link.rel = 'preload'; link.as = 'style'; link.href = href; document.head.appendChild(link); });

Anda dapat menambahkan URL resource penting lainnya ke array preloadLinks jika diperlukan.

‚öôÔ∏è 8.3 Optimasi CSS dan JavaScript

File CSS dan JavaScript sudah di-minify dan digabungkan untuk mengurangi ukuran file dan mempercepat penguraian browser.

Hindari Penambahan Berlebihan

Menambahkan terlalu banyak CSS inline atau JavaScript eksternal yang tidak dioptimalkan dapat memperlambat website Anda.

12. Deploy Website

Setelah semua konfigurasi selesai, saatnya untuk meluncurkan toko online Anda!

üöÄ 12.1 Firebase Hosting

Firebase Hosting adalah cara termudah dan tercepat untuk mendeploy website ini.

  1. Instal Firebase CLI: Jika belum, instal Firebase Command Line Interface (CLI) secara global:
    Instal Firebase CLI
    npm install -g firebase-tools
  2. Login ke Firebase: Buka terminal/command prompt di folder proyek Anda dan login:
    Login Firebase
    firebase login

    Ikuti instruksi di browser untuk otentikasi.

  3. Inisialisasi Proyek: Di folder proyek Anda, inisialisasi Firebase untuk hosting:
    Inisialisasi Firebase Hosting
    firebase init hosting

    Pilih proyek Firebase yang sudah Anda buat. Untuk pertanyaan "What do you want to use as your public directory?", ketik . (titik) karena file `index.html` ada di root. Pilih "No" untuk "Configure as a single-page app (rewrite all urls to /index.html)?".

  4. Deploy Website: Setelah inisialisasi, deploy website Anda:
    Deploy ke Firebase Hosting
    firebase deploy --only hosting

    Setelah selesai, Anda akan mendapatkan URL hosting Firebase Anda.

üåê 12.2 Custom Domain

Untuk menggunakan domain Anda sendiri (misalnya, Femmeia.me):

  1. Buka Firebase Console > Hosting.
  2. Klik "Add custom domain".
  3. Ikuti instruksi untuk memverifikasi domain Anda dan menambahkan catatan DNS (A record atau CNAME record) di penyedia domain Anda.
  4. Firebase akan secara otomatis menyediakan sertifikat SSL untuk domain kustom Anda.

15. Troubleshooting

Berikut adalah beberapa masalah umum dan solusinya.

üõ†Ô∏è 15.1 Masalah Umum

Masalah Penyebab Potensial Solusi
Website tidak tampil setelah deploy Firebase Hosting belum di-deploy dengan benar, atau ada masalah koneksi. Pastikan Anda telah menjalankan firebase deploy --only hosting. Periksa status proyek di Firebase Console.
Gambar produk tidak muncul URL gambar di products array salah, atau gambar belum diunggah ke Storage. Verifikasi URL gambar di index.html. Pastikan gambar sudah diunggah ke Firebase Storage atau GitHub dan URL-nya dapat diakses publik.
Fungsi login/register tidak berjalan Konfigurasi Firebase di index.html salah, atau Authentication belum diaktifkan di Firebase Console. Periksa kembali firebaseConfig di index.html. Pastikan Email/Password Authentication diaktifkan di Firebase Console. Periksa konsol browser untuk error JavaScript.
"Permission denied" di konsol browser Aturan keamanan Firestore atau Storage terlalu ketat. Tinjau kembali firestore.rules dan storage.rules di Firebase Console. Untuk pengembangan, Anda bisa menggunakan "test mode" sementara.
Website lambat Gambar tidak dioptimalkan, terlalu banyak resource, atau masalah jaringan. Optimalkan ukuran gambar. Gunakan Google PageSpeed Insights untuk analisis lebih lanjut.
Fitur Dark Mode/Font Size tidak berfungsi JavaScript yang mengelola pengaturan mungkin tidak dimuat atau ada konflik. Periksa konsol browser untuk error JavaScript. Pastikan tidak ada kode yang menimpa fungsionalitas pengaturan.
